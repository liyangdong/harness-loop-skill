# AGENTS.md — harness-loop skill

> Conventions for editing this skill. Agents working inside `.claude/skills/harness-loop/`
> MUST follow this file. It mirrors the ≤100-line constraint this skill imposes on its
> generated outputs (dogfood rule, see §6).

## 1. Skill purpose

This skill generates AGENTS.md-based constraint systems via an 8-question wizard. When
invoked in any project, it asks 8 questions, then writes a complete methodology-aware
AGENTS.md plus the supporting directory scaffold, check scripts, and templates.

## 2. Map-not-manual

- `SKILL.md` stays ≤100 lines: it lists triggers, entry points, and pointers only.
- All prose detail lives in `wizard/` (the 8 questions and their branching logic).
- All reusable artifacts live in `templates/` (methodologies, concepts, checks, scaffolding).
- This AGENTS.md stays ≤100 lines and is the single source of truth for *how to edit the skill*.

## 3. Editing rules

| Change | Where to edit |
|---|---|
| Add a methodology (e.g. BDD) | New file `templates/methodologies/<name>.md` + new entry in `wizard/decision-tree.md` Q3 |
| Add a check layer (e.g. perf) | New file `templates/checks/check-<name>.sh.tmpl` + entry in `wizard/decision-tree.md` Q7 |
| Add a language (e.g. rust) | New directory `templates/scaffolding/methodology-dirs/tests-rust/` (mirror an existing `tests-<lang>/`) |
| Add a concept (e.g. async) | New file `templates/concepts/<name>.md` |
| Tweak a wizard question | Edit `wizard/questions.md` and `wizard/decision-tree.md` together — keep them in sync |

Do **not** bloat `SKILL.md` with option lists. If a new option needs more than a one-line
pointer, it belongs in `wizard/` or `templates/`.

## 4. Naming conventions

- Directories: lowercase-kebab-case (`tests-java`, `readme-first`).
- Template files: `<name>.md.tmpl` for prose, `<name>.sh.tmpl` for shell.
- Subdir `AGENTS.md` files generated by the skill MUST stay ≤100 lines.

## 5. Self-verification

Before committing any change to this skill, run:

```bash
bash .claude/skills/harness-loop/scripts/check-skill.sh
```

This verifies: every directory listed in `templates/scaffolding/methodology-dirs/` has at
least one template file, `wizard/questions.md` and `wizard/decision-tree.md` are in sync,
and no generated sample exceeds the 100-line rule. Do not commit on a failing check.

## 6. Dogfood

This file itself follows the same ~100-line rule it imposes on generated `AGENTS.md`
outputs. If you must exceed it, split detail into `wizard/` or `templates/` and leave a
pointer here.
