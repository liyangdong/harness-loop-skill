# scripts/

Mechanical enforcement scripts. One file per concern, composed by
`.githooks/pre-commit`. Docs rot, lint rules don't (Concept 03).

## 本目录约定

- `check-consistency.sh` — C1/C2/C6: number consistency, subdir AGENTS.md
  presence, methodology-required dirs. Generated by harness-loop.
- Each check is one file, one concern. No mega-`check-all.sh`.
- Checks must be deterministic: same repo state → same exit code.
- Error messages embed fix instructions: file path, line, what's wrong, how
  to fix.
- Strict mode (this repo): failures exit non-zero and block commit/merge.

## 与根 AGENTS.md 的关系

继承根 AGENTS.md 的 6 大概念（特别是 Concept 03: Mechanical Enforcement）。
本文件只补充本目录特有规则。

## 验证

本目录相关的检查：
- `.githooks/pre-commit` — calls every `check-*.sh` in this directory when
  controlled paths are staged.
- `.github/workflows/consistency.yml` — mirrors pre-commit on push/PR.
